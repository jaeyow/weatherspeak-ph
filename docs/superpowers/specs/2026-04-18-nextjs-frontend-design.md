## Overview

A mobile-first Next.js PWA deployed on Vercel that lets Filipino users listen to PAGASA typhoon bulletins translated into Tagalog, Cebuano, and English. Audio is pre-generated by the Modal ETL pipeline and stored in Supabase Storage. The frontend is a read-only consumer of that data — no auth, no write operations.

---

## Architecture

### Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 14 App Router, TypeScript |
| Styling | Tailwind CSS |
| Data | `@supabase/ssr` — server components + ISR |
| Deployment | Vercel |
| Storage | Supabase Storage (public bucket `weatherspeak-public`) |
| State | localStorage only (no auth, no server-side session) |

### Rendering strategy

- **ISR (Incremental Static Regeneration)**: pages are pre-rendered and served from Vercel's CDN. Supabase is queried in the background to regenerate stale pages.
  - Home page: revalidate every 15 minutes
  - Storm + bulletin pages: revalidate every 10 minutes
- **Server components** query Supabase with the service-role key (server-side only, never exposed to client)
- **Client components** use the public anon key where needed (safe — RLS enforces public read-only)
- Audio and chart files stream directly from Supabase Storage public URLs — no proxying

### ETL trigger

Manual only for the hackathon. Operator runs `uv run modal run modal_etl/run_batch.py` when a new PAGASA bulletin is published. ISR picks up the new Supabase data within 15 minutes with no further action required.

## Routes

| Route | Rendering | Purpose |
|---|---|---|
| `/` | ISR 15 min | Active storms list + past storms archive + onboarding |
| `/storms/[stormId]` | ISR 10 min | Storm detail — hero + latest bulletin audio |
| `/bulletins/[bulletinId]` | ISR 10 min | Individual bulletin audio + chart + affected areas |

No custom API routes. All Supabase access happens in server components.

## Pages

### `/` — Home

**Active storms section (top)**
- Fetches `storms_with_status` view: `WHERE is_active = true ORDER BY current_signal DESC, last_bulletin_at DESC`
- Each storm rendered as a **signal-first card**:
  - Left: signal block (large number, colour-coded: 4–5 = red `#c0392b`, 3 = orange, 2 = yellow, 1 = blue)
  - Right: storm name, category, current reference position, distance from user (client-side haversine), last updated time
- Empty state: calm status banner — "No active typhoons right now. Stay prepared." — not a blank page
- Tapping a card navigates to `/storms/[stormId]`

**Past storms section (always visible below active)**
- Fetches `storms_with_status`: `WHERE is_active = false ORDER BY last_bulletin_at DESC LIMIT 20`
- Smaller cards: storm name, peak signal, date range, bulletin count
- Gives the demo something to show when no typhoon is active

**First-visit onboarding modal**
- Triggered when `ws_province` is absent from localStorage
- Single screen: province dropdown → city/municipality dropdown (cascading, driven by `philippines-geography.json`) + language pill selector (Tagalog / Cebuano / English) + "Get Started" button
- On submit: writes `ws_province`, `ws_city`, `ws_lat`, `ws_lon`, `ws_language` to localStorage, dismisses modal
- Skipped on subsequent visits

**Header (persistent)**
- App name + typhoon icon
- Global language toggle pill: `TL · CEB · EN` — client component, writes `ws_language` to localStorage
- Location display: "📍 Cebu City" — links to re-open location settings

---

### `/storms/[stormId]` — Storm Detail

Displays the latest bulletin for the storm. Older bulletins are listed below.

**Hero banner** (colour matches signal level)
- Storm name (large), category, signal level
- Current reference position ("850 km east of Catanduanes")
- Distance pill: "📍 320 km from Cebu City" (client-side, from localStorage coords)

**Audio player block**
- Shows audio for the globally-selected language (`ws_language`)
- Track label: language name + bulletin number + duration
- Progress bar (scrubable), play/pause button
- Download button: triggers browser download of the MP3 from Supabase Storage
- If `bulletin_media.status !== 'ready'` for the selected language: "Audio not yet available"

**Storm track chart**
- `next/image` with blur placeholder
- Source: `chart_path` from the latest bulletin row in Supabase Storage
- Null-safe: hidden if `chart_path` is null

**Affected areas accordion**
- Sections: Signal 4 / Signal 3 / Signal 2 / Signal 1 — each expandable
- Data from `affected_areas` JSONB on the bulletin row

**Past bulletins list**
- All bulletins for this storm, newest first, excluding the latest (already shown above)
- Each row: bulletin number, issued_at, link → `/bulletins/[bulletinId]`

**Mode 2 stub** (bottom of page)
- Greyed-out card: "Storm Summary Audio — Coming Soon"
- No functionality — signals future roadmap

---

### `/bulletins/[bulletinId]` — Bulletin Detail

Same layout as the storm detail audio section, applied to a specific (potentially older) bulletin.

- Back button → `/storms/[stormId]`
- Hero, audio player, chart, affected areas — all from that bulletin's data
- No past bulletins list (user navigated here from the storm page list)

## Key Components

### Client components

**`<LanguageToggle>`**
- Reads/writes `ws_language` in localStorage
- Pill selector: TL · CEB · EN
- On change: writes to localStorage and dispatches a `ws:language-change` custom window event so `<BulletinAudioSection>` can react without a shared state library
- Lives in the header

**`<BulletinAudioSection>`**
- Client wrapper that bridges the server-rendered bulletin data and the language toggle
- Props: `media: Record<"en"|"tl"|"ceb", { audioPath: string | null, durationSeconds: number | null, status: string }>`
- Reads `ws_language` from localStorage on mount and re-reads whenever `<LanguageToggle>` fires a custom event
- Picks the right media object and renders `<AudioPlayer>` with the resolved URL
- This is necessary because the server component (ISR) cannot react to localStorage changes — a client component must own that logic

**`<AudioPlayer>`**
- Props: `audioUrl: string | null`, `durationSeconds: number | null`, `filename: string`
- Handles play/pause, progress bar (seek), time display
- Download: `<a href={audioUrl} download={filename}>` — native browser download
- Disabled/loading state when `audioUrl` is null

**`<LocationOnboarding>`**
- Modal overlay shown on first visit
- Province dropdown populated from `philippines-geography.json`
- City dropdown cascades from selected province
- Language pills: Tagalog / Cebuano / English
- On submit: writes 5 localStorage keys, closes modal

**`<DistancePill>`**
- Reads `ws_lat`, `ws_lon` from localStorage
- Receives `stormLat: number`, `stormLon: number` as props
- Computes haversine distance client-side
- Renders "📍 {distance} km from {ws_city}" or nothing if location not set

**`<SignalBadge>`**
- Props: `signal: number | null`
- Colour-coded square block: 4–5 red, 3 orange, 2 yellow, 1 blue, null grey
- Used in cards and hero banner

### Server components

**`StormList`** — queries `storms_with_status`, renders active and past storm card grids

**`StormDetail`** — queries storm + latest bulletin + `bulletin_media` × 3 languages + bulletin history list

**`BulletinDetail`** — queries single bulletin + `bulletin_media` × 3 languages

## Data Layer

### Supabase queries

All queries run in Next.js server components using the service-role key via `@supabase/ssr`.

**Home — active storms**
```sql
SELECT * FROM storms_with_status
WHERE is_active = true
ORDER BY current_signal DESC NULLS LAST, last_bulletin_at DESC;
```

**Home — past storms**
```sql
SELECT * FROM storms_with_status
WHERE is_active = false
ORDER BY last_bulletin_at DESC
LIMIT 20;
```

**Storm detail**
```sql
-- Storm row
SELECT * FROM storms WHERE id = $stormId;

-- Latest bulletin + media
SELECT b.*, bm.language, bm.audio_path, bm.script_path, bm.tts_path,
       bm.audio_duration_seconds, bm.status
FROM bulletins b
LEFT JOIN bulletin_media bm ON bm.bulletin_id = b.id
WHERE b.storm_id = $stormId
ORDER BY b.issued_at DESC;
```

**Bulletin detail**
```sql
SELECT b.*, bm.language, bm.audio_path, bm.audio_duration_seconds, bm.status,
       b.chart_path, b.affected_areas
FROM bulletins b
LEFT JOIN bulletin_media bm ON bm.bulletin_id = b.id
WHERE b.id = $bulletinId;
```

### Audio URL construction

```ts
const audioUrl = (audioPath: string) =>
  `${process.env.NEXT_PUBLIC_SUPABASE_URL}/storage/v1/object/public/weatherspeak-public/${audioPath}`;
```

### Philippines geography

Static JSON file at `data/philippines-geography.json`, bundled with the Next.js app. Structure:

```json
{
  "provinces": {
    "Cebu": { "lat": 10.3157, "lon": 123.8854, "region": "Region VII" }
  },
  "cities": {
    "Cebu City": { "province": "Cebu", "lat": 10.3157, "lon": 123.8854 }
  }
}
```

Covers all 81 provinces and ~50 major cities. One-time manual compilation from PSA 2020 census data + Wikipedia coordinates. **This file does not exist yet and must be created before the onboarding modal can function.**

### Distance calculation

```ts
function haversine(lat1: number, lon1: number, lat2: number, lon2: number): number {
  const R = 6371;
  const dLat = ((lat2 - lat1) * Math.PI) / 180;
  const dLon = ((lon2 - lon1) * Math.PI) / 180;
  const a =
    Math.sin(dLat / 2) ** 2 +
    Math.cos((lat1 * Math.PI) / 180) *
    Math.cos((lat2 * Math.PI) / 180) *
    Math.sin(dLon / 2) ** 2;
  return Math.round(R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a)));
}
```

## Client-side State (localStorage)

| Key | Type | Purpose |
|---|---|---|
| `ws_language` | `"en" \| "tl" \| "ceb"` | Global language toggle |
| `ws_province` | `string` | Selected province name |
| `ws_city` | `string` | Selected city name (display) |
| `ws_lat` | `string` (numeric) | City latitude for distance calc |
| `ws_lon` | `string` (numeric) | City longitude for distance calc |

If `ws_province` is absent → show onboarding modal.
If `ws_lat`/`ws_lon` are absent → `<DistancePill>` renders nothing.

## Error Handling

| Scenario | Behaviour |
|---|---|
| Supabase fetch fails | Static error page ("Unable to load storm data"). ISR-cached pages remain available from CDN. |
| `audio_path` null / status not ready | Audio player hidden, "Audio not yet available" message shown |
| `chart_path` null | Chart section hidden |
| Location not set | `<DistancePill>` renders nothing; rest of page unaffected |
| No active storms | Calm banner + past storms archive fills the page |

## Out of Scope (Hackathon)

- User authentication
- Service worker / offline caching (download button covers this)
- ETL cron trigger (manual `modal run`)
- Mode 2 storm summary synthesis (stub only)
- Push notifications
- Native Bisaya TTS (MMS fallback already in place)
- More than 3 languages
